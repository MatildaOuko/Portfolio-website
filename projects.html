Network Threat Analysis and Detection

Objective:
Simulate network activity through a port scan and analyse captured traffic using Wireshark in order to identify reconnaissance behaviour.

Traffic Capture:

Tools Used:

  • Nmap
  • Wireshark
  • Local machine (Kali Linux)

Methodology:

  • Launched Wireshark and configured it to capture packets on the active network interface
  • Performed a TCP SYN (stealth) scan using: nmap -sS localhost
  • Applied filter to isolate SYN packets

Attack Simulation:

Proof of Concept:

Applied filter tcp.flags.syn == 1 && tcp.flags.ack == 0 to isolate SYN packets.

Traffic from 192.168.93.128 shows repeated SYN requests to multiple destinations over port 443, indicating incomplete TCP handshakes consistent with port scanning.

Filtered Traffic Analysis:

Explanation:

This behaviour demonstrates reconnaissance activity where a system scans multiple hosts to identify open ports and services.

Automated Lab Analysis

Objective:
Automate the analysis of log data to reduce manual effort and quickly identify important events.

Raw Log Data:

Problem:
Manual analysis of logs is time-consuming and inefficient.

Approach:

  • Developed Python script to process log files
  • Filtered entries using keywords (ERROR, WARNING)
  • Generated structured output

Tools Used:

  • Python

Automation Script:

Proof of Concept:

for line in lines:
    if "ERROR" in line:
        errors.append(line)
    elif "WARNING" in line:
        warnings.append(line)

The script successfully identified:

  • Failed login attempts
  • Connection timeouts
  • High memory usage warnings

Script Output:

Security Insight:

Failed login attempts may indicate brute-force attacks, demonstrating how automation supports threat detection.

Results:

  • Reduced analysis time
  • Improved accuracy
  • Faster detection of issues

What I Learned:

  • Log parsing using Python
  • Automation of repetitive tasks
  • Security relevance of log analysis
